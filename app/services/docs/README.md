# 레시피 추천 시스템 문서 인덱스 (Cheat Sheet)

이 디렉토리는 레시피 추천 시스템의 성능 최적화 및 UX 개선 프로젝트와 관련된 핵심 문서들을 담고 있습니다. 개발 및 유지보수 시 아래 가이드를 참고하여 필요한 정보를 확인하세요.

## 📂 문서 목록

| 파일명 | 목적 및 내용 | 주요 대상 독자 |
| :--- | :--- | :--- |
| **[want.md](./want.md)** | **사용자 요구사항 정의서**<br>- 최종적으로 구현되어야 할 UX/UI, 성능, 안전성 목표<br>- LLM 상호작용의 대원칙 (판단 지양, 실행 지향) | 기획자, 개발자, QA |
| **[progress_record.md](./progress_record.md)** | **진행 상황 기록 (Change Log)**<br>- 초기 시도(Mock)부터 현재까지의 의사결정 과정 및 변경 이력<br>- 실패 사례와 그 원인 분석 포함 | 개발자 |
| **[finding.md](./finding.md)** | **구현 검토 및 회고 (Self-Reflection)**<br>- 현재 구현 상태에 대한 자체 평가<br>- 놓친 부분, 보완해야 할 점, 향후 과제(Next Steps) | 개발자 |
| **[temp_sum.md](./temp_sum.md)** | **(임시) 구현 가이드 요약**<br>- 실제 코딩에 들어가기 직전에 정리한 구체적인 구현 스펙<br>- 파일 경로, 함수명, 로직 흐름 등이 상세히 기술됨 | 개발자 |

## 💡 개발 시 참고 가이드

1.  **"내가 지금 뭘 만들고 있지?"** 헷갈릴 때 👉 **`want.md`**
    *   사용자가 원했던 핵심 가치(속도, 건강 체크, 선택권)를 다시 상기하세요.

2.  **"왜 이렇게 구현했지?"** 의문이 들 때 👉 **`progress_record.md`**
    *   과거의 시행착오와 전략 수정 과정을 확인하여 현재 아키텍처의 배경을 이해하세요.

3.  **"이제 뭐 해야 하지?"** 다음 스텝이 궁금할 때 👉 **`finding.md`**
    *   남은 과제와 보완점을 확인하고 우선순위를 정하세요.

4.  **"어떻게 짜야 하지?"** 구체적인 코드가 필요할 때 👉 **`temp_sum.md`** (또는 소스 코드 직접 확인)
    *   구현 로직과 프롬프트 전략을 참고하세요.
