## Issue1: Meals/Ingredients API Always Return `user_id = 1`

### 상황 요약
- 프론트엔드에서 식단 및 식재료 API를 호출할 때 응답의 `user_id`가 항상 `1`로 내려왔다.
- 실제 로그인한 사용자와 무관하게 같은 ID가 사용돼, 모든 사용자가 한 사람의 데이터만 조회/수정하는 문제가 발생했다.

### 근본 원인
1. `app/api/v1/routes/meals.py`와 `app/api/v1/routes/ingredients.py`에 임시 함수 `get_current_user_id()`가 정의돼 있었고, 이 함수가 **하드코딩으로 `1`을 반환**했다.
2. 세션 의존성(`require_authentication`)을 사용하지 않아 FastAPI가 HTTP 세션에서 로그인 사용자 정보를 읽어올 기회가 없었다.

### 해결 과정
1. 두 라우터에서 임시 `get_current_user_id()` 함수를 전부 제거했다.
2. 각 파일 상단에 `from app.api.dependencies import require_authentication`를 추가했다.
3. 모든 엔드포인트 함수 시그니처에 `user_id: int = Depends(require_authentication)`를 주입하고, 기존의 하드코딩된 사용자 ID 사용처를 이 값으로 대체했다.
4. 이제 FastAPI가 세션에서 인증된 사용자의 `user_id`를 주입하므로, 프론트엔드에서도 로그인한 사용자 ID가 그대로 전달된다.
